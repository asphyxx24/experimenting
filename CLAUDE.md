# Repo: Ideen-Sammlung

Dieses Repository ist ein Sammelbecken für Projekt- und Produkt-Ideen. Es ist kein produktives Projekt — jede Idee lebt in einem eigenen Unterordner und kann unabhängig wachsen (von reiner Recherche bis hin zu Code-Prototypen).

## Layout

- `ideas/` — Eine Idee pro Unterordner.
  - `ideas/README.md` — Index aller Ideen mit Status.
  - `ideas/<name>/README.md` — Details, Recherche, Entscheidungen, offene Fragen.
  - `ideas/<name>/CLAUDE.md` (optional) — projekt-spezifische Anweisungen, falls die Idee einen eigenen Tech-Stack hat.
  - `ideas/<name>/*` — Code, Prototypen, Skizzen, etc.

## Arbeitsweise

- Wenn der User über eine konkrete Idee redet, **erst die README im jeweiligen Idee-Ordner lesen**, bevor du Annahmen triffst.
- Neue Idee → neuen Unterordner anlegen und in `ideas/README.md` eintragen.
- Status-Updates an einer Idee auch im Index pflegen, damit der Überblick stimmt.
- Branches: pro Idee ein eigener Branch ist okay, aber für reine Recherche-Updates reicht oft auch direkt `master`.

## Sprache

Antworten auf Deutsch. Code-Identifier und technische Begriffe in ihrer Originalform.
